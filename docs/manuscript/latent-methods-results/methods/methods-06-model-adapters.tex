\subsection{Model Adapter Architecture}
\begin{itemize}
  \item \texttt{ModelAdapter} ABC: \texttt{forward(B,C,H,W,D) $\to$ SegmentationOutput}
  \item Three abstract methods required per model:
    \begin{enumerate}
      \item \texttt{forward()} --- model-specific inference
      \item \texttt{get\_eval\_roi\_size()} --- patch size for sliding window inference
      \item \texttt{get\_adapter\_config()} --- hardware profile metadata
    \end{enumerate}
  \item \texttt{build\_adapter(config)} factory for config-driven dispatch
  \item Current implementations: DynUNet, SegResNet, SAM3 Vanilla/Hybrid/TopoLoRA,
        VesselFM, SwinUNETR, AttentionUNet
  \item Adding a new model = implement 3 methods + 1 YAML profile
        (\texttt{configs/model\_profiles/model\_name.yaml})
\end{itemize}
