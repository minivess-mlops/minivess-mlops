\subsection{Configuration Architecture}
\begin{itemize}
  \item Dual configuration system: Hydra-zen (experiment sweeps) + Dynaconf (deployment environments)
  \item \texttt{.env.example} = single source of truth for ALL service URLs, ports, and credentials
  \item Zero hardcoded values --- any lab overrides via \texttt{configs/lab/lab\_name.yaml}
  \item Config hierarchy: repo defaults $\to$ lab override $\to$ user override $\to$ experiment
  \item Config groups: \texttt{configs/cloud/}, \texttt{configs/registry/}, \texttt{configs/model\_profiles/}
  \item New lab adoption: create \texttt{configs/lab/newlab.yaml} + \texttt{configs/user/user.yaml}
        --- zero code changes required
\end{itemize}
