### Task 1 (P0): Dynamic ordered: injection in run_factorial.sh

**The core implementation task.**

`scripts/run_factorial.sh` currently does NOT read `region_config` from the factorial YAML.
It passes the static `deployment/skypilot/train_factorial.yaml` directly to `sky jobs launch`.

**Required logic** (pseudo-code):
```bash
# 1. Read region_config name from factorial YAML
REGION_CONFIG_NAME=$(python3 -c "
import yaml, pathlib
cfg = yaml.safe_load(pathlib.Path('.../${CONFIG_FILE}').read_text(encoding='utf-8'))
print(cfg.get('infrastructure', {}).get('region_config', ''))
")

# 2. If region_config is set, load the region file
REGION_CONFIG_PATH="${REPO_ROOT}/configs/cloud/regions/${REGION_CONFIG_NAME}.yaml"

# 3. Generate temp SkyPilot YAML with ordered: injected
#    Read base YAML, extract regions for the GPU type (L4),
#    build ordered: list, write to temp file
TEMP_SKYPILOT_YAML=$(python3 -c "
import yaml, pathlib, tempfile
base = yaml.safe_load(pathlib.Path('${SKYPILOT_YAML}').read_text(encoding='utf-8'))
regions = yaml.safe_load(pathlib.Path('${REGION_CONFIG_PATH}').read_text(encoding='utf-8'))

# Extract L4 regions for GCP
l4_regions = regions['regions']['gcp']['L4']

# Build ordered: list
ordered = [{'accelerators': 'L4:1', 'region': r['region']} for r in l4_regions]

# Inject into resources
base['resources']['ordered'] = ordered
# Remove the default accelerators: L4:1 (ordered: supersedes it)
base['resources'].pop('accelerators', None)

# Write temp file
tmp = tempfile.NamedTemporaryFile(suffix='.yaml', delete=False, mode='w')
yaml.dump(base, tmp, default_flow_style=False)
tmp.close()
print(tmp.name)
")

# 4. Use temp YAML for sky jobs launch (instead of static SKYPILOT_YAML)
sky jobs launch "${TEMP_SKYPILOT_YAML}" --name "${CONDITION_NAME}" ...

# 5. Clean up temp file after all launches complete
```

**Critical constraints**:
- The temp YAML must pass `sky.Task.from_yaml()` validation
- When `region_config` is empty/missing (e.g., smoke_local.yaml), use the static YAML as-is
- The `accelerators: L4:1` line in the base YAML is a FALLBACK for dry-run mode;
  when `ordered:` is injected, `accelerators:` should be removed from resources
- Use Python's `yaml.dump()` for generation -- NO string templates, NO f-strings for YAML
- The temp file approach avoids modifying the committed `train_factorial.yaml`
